**Date:** 2026-02-24

## Service Worker Caching Strategy

Using Workbox via `@vite-pwa/astro`. Precaches app shell automatically (JS/CSS chunks with hash). Runtime caching rules:

| Resource | Strategy | TTL | Reason |
|----------|-----------|-----|--------|
| App shell (JS/CSS/fonts) | `CacheFirst` | Precached w/ hash | Never changes between deploys |
| API `/api/events/**` | `StaleWhileRevalidate` | 24h max-age | Show cached instantly, refresh in background |
| S3 images/thumbnails (`*.amazonaws.com`) | `CacheFirst` | 7 days | Photos don't change once uploaded |
| S3 videos | Not cached | вЂ” | Too large for device storage |
| Google Maps | `NetworkFirst` | вЂ” | Always needs live data |

**Offline fallback:** If API call fails and no cache exists в†’ SW returns `/offline` page (precached static page).

## Install Prompt Component

**File:** `src/components/InstallPrompt.tsx`

Smart behavior:
- **Android Chrome**: Intercepts `beforeinstallprompt`, defers it, shows custom banner after 3s delay on event pages (`/e/[identifier]`). Never on root `/`.
- **iOS Safari**: Detects `navigator.standalone === false` + iOS userAgent в†’ shows tooltip "Toca Compartir в†’ Agregar a inicio" with animated share icon.
- **Already installed**: `window.matchMedia('(display-mode: standalone)').matches` в†’ renders nothing.
- **Dismissed**: Stores `pwa-install-dismissed` timestamp in localStorage. Hides for 7 days.
- **Second visit**: Shows again after 7-day cooldown.

UI: Slides in from bottom. Pink eventiapp branding. Close button. On Android: "Instalar app" button triggers native prompt. On iOS: instructional tooltip with share icon illustration.

Mounted as `client:only="react"` inside `EventPage` so it only appears on event invitation pages.
